통신과 비동기식 통신

  • 통신 : 어떤 것을 주고 받는 행위
  • http 통신 : html 문서를 요청하고 받는 행위
  • 브라우저의 통신 : http 통신
  • 브라우저의 동기식 통신: 브라우저의 주소창에서 주소 html 문서를 요청하고 html 문서를 받으면 화면이 s 해당 페이지로 전환되는 것
  • iframe 이용한 비동기식 통신: iframe 태그 안에서 다른 페이지(html)를 불러오는 것
  • 브라우저의 비동기식 통신(AJAX): html 문서에서 다른 페이지(xml,html,json,text)를 불러오기위해 http 통신을 하는 것

XMLHttpRequest 객체와 비동기식 통신

  • html 문서 안에서 다른 페이지를 호출하고 처리하기 위해 사용하는 객체
  • XMLHttpRequest.open(method,url,async(defult:ture)) : 요청할 페이지를 준비
  • XMLHttpRequest.onreadystatechange=(callback func) : 통신하면서(이벤트) 실행되는 함수 (실패해도 실행)
  • XMLHttpRequest.send() : 실제로 통신해서 페이지를 불러온다.

XMLHttpRequest.status == HTTP response status codes

통신의 응답 상태로 모든 http 통신에서 사용되는 코드다.

http 응답 상태 코드는 http 요청이 성공적인지 확인하는 코드로 총 5단계로 나뉜다.

  • 100~199 : 정보 응답
  • 200~299 : 성공적인 응답 (200)
  • 300~399 : 메세지
  • 400~499 : 클라이언트오류 (400,404)
  • 500~599 : 서버 오류
  • 200 (OK):
    • GET : 응답 리소스를 받아서 바디에 전송
    • POST : 요청 해더에 메세지(파라미터)를 전송한다. 요청을 처리한다.(db에 저장 삭제 수정)
  • 404 : 요청하는 페이지가 없음
  • 400 : 요청하는 방식이 잘못되었음(method)

XMLHttpRequest.readyState : 요청의 처리상태

XMLHttpRequest는 통신을 위해 총 5단계의 실행을 거친다.

  • 0(UNSENT) : 미전송 상태로 new로 XMLHttpRequest 객체를 생성하기만 한 상태
  • 1(OPENED) : open 메소드를 실행한 상태로 요청 헤더(setRequestHeader()) 사용할 수 있고 **send()로 요청 할 수 있다.
  • 2(HEADERS_RECEIVED) : send()호출하고 응답 해더가 수신(onreadystatechange가 실행)
  • 3(LOADING) : 본문을 수신하고 해석하는 중(onreadystatechange가 실행)
  • 4(DONE) : 응답 완료한 상태
