자바스크립트의 함수

함수는 프로그래밍 언어에서 가장 중요한 실행을 담당한다. 함수는 실행의 그룹화된 가장 작은 단위로 보통은 사용되나 함수지향 언어인 자바 스크립트에서는 클래스 처럼 사용되기도 한다.

자바스크립트의 실행은 순서를 따르지만 함수는 어디에 선언해도 참조할 수 있다.

자바 스크립트는 같은 이름의 함수를 (파라미터가 갯수가 다른) 여러개 가질수 없다. 때문에 함수를 선언할때 상수에 익명함수를 참조하는 형식이 좋다. (const a = function(){}, const a=function(a,b){} ->오루)

기본 선언법: function name(parameter) {return a;}

변수에 익명함수 선언법 - 함수표현식 (Function Expression): const name = function(parameter){return a;}

익명함수란: 인스턴스 즉 쓰고 사라질 함수로 보통은 매개변수로 쓰일때 가볍게 선언하려 만들어진다.
예) const a=function(){}:익명함수지만 변수가 함수의 이름을 대신한다.
document.onclick = function(){}이때 onclick을 재정의한 익명함수는 이벤트시 발생하고 사라진다.
document.onclick = a;

es6에서 추가된 화살표함수(익명함수로 사용하기를 권장) : const a = (parameter)=>{return a;}
예) document.onclick=()=>{};

  • document.addEventListner("onclick",function(){});
  • document.addEventListner("onclick",()=>{});
  • window.setInerVal(function(){},1000);
  • window.setInerval(()=>{},1000);

    • const a=function(a,b=default): 함수의 매개 변수는 default 값을 가질수 있다. 이때 a 는 function(a), function(a,b) 둘다를 선언한것과 같으니오버로딩과 같다.

    const a=function(...args){}; a(1,2,3,4,5,6...); ...args는 함수의 배개변수 개수를 제한하지 않는다. 이때 args는 배열 타입이다.

    자바스크립트는 함수를 객체로 인식한다. (인스턴스 객체로 생성하는 함수는 권장하지 않는다.)
